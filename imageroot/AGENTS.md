# imageroot Guidelines

- This subtree is the installed NS8 module payload. Keep it aligned to the checked-in per-agent Hermes runtime, which now runs the dashboard and gateway together, plus the shared auth publishing service.
- There is no OpenViking runtime, no hidden backend runtime, and no `AGENTS_LIST` registry anymore. There is now one shared auth publishing target: `hermes-auth.service` plus the generated `authproxy.env`, `authproxy_secrets.env`, and `authproxy_agents.json` files.
- The runtime contract is now: one configured agent equals one metadata file, one generated Hermes env file, one generated Hermes secrets env file, one Podman-managed Hermes home volume, one `hermes@<id>.service`, one `hermes-socket@<id>.service`, one `hermes-pod-<id>` pod, one rootless `hermes-<id>` container that runs the dashboard and gateway together, one rootless `hermes-socket-<id>` relay container, and one dashboard socket file at `dashboard-sockets/agent-<id>.sock`. Shared publishing adds one `hermes-auth.service`, one rootless `hermes-auth` container, one shared Traefik route, and the generated shared auth runtime files.
- Keep restart ownership in systemd: `hermes@<id>.service` and `hermes-auth.service` handle restart policy, and the Podman pod/container launches should not add container-level `--restart` policies.
- Preserve named-volume ownership semantics: keep one Hermes home volume per agent, and do not regress the runtime into shared or root-owned agent homes across restarts. Use `ensure-agent-home-ownership` so ownership follows the configured Hermes image's dynamic `hermes` UID/GID instead of a hardcoded numeric UID. `update-module.d/30ensure-agent-home-ownership` owns the best-effort stop, reset, and repair flow, and `update-module.d/80restart` owns restarting the enabled Hermes, socket, and shared auth services afterward.
- Preserve the NS8 action model already used here: numbered executable action steps, JSON stdin for actions, JSON stdout for reads, and schema files beside the actions.
- `environment` is shared NS8 state. Merge only managed keys and preserve core-managed values such as `HERMES_AGENT_HERMES_IMAGE`, `HERMES_AGENT_AUTH_IMAGE`, and `HERMES_AGENT_SOCKET_IMAGE`.
- Keep module-wide secrets in `secrets.env`. Keep generated per-agent Hermes secrets in `agent_<id>_secrets.env`, and keep shared auth runtime outputs in `authproxy.env`, `authproxy_secrets.env`, `authproxy_agents.json`, and `dashboard-sockets/`.
- Keep shared smarthost discovery in `discover-smarthost`, keep per-agent env and secrets generation in `sync-agent-runtime`, and keep strict first-seed-only home seeding in `configure-module/75seed-agent-home`. `sync-agent-runtime` owns the generated `authproxy_agents.json` `upstream_socket` records. That seed step must use the generated public `agent_<id>.env`, mount checked-in templates read-only, and preserve existing `SOUL.md` plus `.env` content in the volume.
- `sync-agent-runtime` now owns the generated `AGENT_ALLOWED_USER` field plus `USER_DOMAIN`, `LDAP_HOST`, `LDAP_PORT`, `LDAP_BASE_DN`, `LDAP_SCHEMA`, `LDAP_BIND_DN`, and `LDAP_BIND_PASSWORD` when shared dashboard auth settings are configured.
- Shared dashboard publishing is controlled by `base_virtualhost`, shared `lets_encrypt`, shared `user_domain`, and per-agent `allowed_user`. If you change route payload fields or cleanup behavior, update Traefik-facing actions, tests, UI, and docs together.
- Do not inject the shared `environment` file directly into containers. Per-agent Hermes containers should consume the generated per-agent env files only, and the shared auth container should consume the generated shared auth env and registry files plus the mounted `dashboard-sockets/` directory only.
